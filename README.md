# SNGmed Landing

Одностраничный лендинг для компании SNGmed, специализирующейся на параллельном импорте инструментария и расходных материалов для робота-ассистента da Vinci Si / IS300.

## Структура проекта
```
.
├─ index.html
├─ assets/
│  ├─ css/style.css
│  └─ js/main.js
├─ images/
│  └─ instruments/  # изображения инструментов (по артикулам)
├─ logo.png
└─ README.md
```

## Локальный запуск
1. Склонируйте репозиторий и перейдите в папку проекта.
2. Откройте `index.html` в любом современном браузере (двойной клик или через локальный веб-сервер).

## Как отправить проект на GitHub
Ниже пример базовой последовательности команд. Замените `USERNAME` и `REPO` на свои значения.

```bash
git init                    # инициализируем локальный репозиторий
git add .                   # добавляем все файлы
git commit -m "Initial commit"
git branch -M main          # переименовываем основную ветку (если нужно)
git remote add origin git@github.com:USERNAME/REPO.git
git push -u origin main     # отправляем проект в GitHub
```

После первого пуша можете продолжать работу в этой же ветке (`main`) или создавать новые ветки под задачи.

## Деплой на GitHub Pages
1. Создайте репозиторий на GitHub и отправьте в него содержимое проекта (`git push origin main`).
2. В настройках репозитория (Settings → Pages) выберите ветку `main` и корневую директорию (`/`).
3. Сохраните настройки — GitHub Pages развернёт сайт и выдаст ссылку вида `https://<username>.github.io/<repo>/`.

## Каталог инструментария
Каталог на странице генерируется из массива `instruments` в `assets/js/main.js`. Чтобы добавить новый инструмент, просто расширьте этот массив (или подключите внешний источник данных) и разместите изображение в `images/instruments/` с именем, совпадающим с артикулом.

## Обработка форм
Форма заявки в секции «Контакты» работает в демо-режиме: при отправке данные выводятся в консоль браузера. Чтобы подключить реальную отправку формы, замените обработчик в `assets/js/main.js` на обращение к вашему backend или почтовому сервису.
